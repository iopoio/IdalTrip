축제 + 관광공사 코스 통합 방법

이달여행 정체성 유지하면서 코스 데이터 활용하기

A

축제 고정 + 코스는 "주변 장소 추천" 역할

축제는 항상 상단 체크 고정 (필수). 코스 서브장소는 "이 축제 근처 관광공사 추천 장소"로 표현. 역할 분리가 명확함.

🎪 속초 벚꽃축제 — 필수 포함
이 축제 근처 관광공사 추천 장소
✅ 속초 등대전망대
✅ 엑스포타워

장점

  • 이달여행 정체성 유지
  • 구현 단순
  • 사용자 혼란 없음

단점

  • 코스 "테마" 느낌 약함
  • 코스 선택지가 의미 작음
B

축제를 코스에 끼워넣기 (지리적 필터)

축제 좌표 반경 20km 내 코스 장소만 표시. 코스 서브장소 + 축제를 묶어서 "이 지역 힐링코스 + 이달 축제" 조합으로 표현.

힐링코스 · 이달 축제 포함
🎪 속초 벚꽃축제 (이달 한정)
✅ 속초 등대전망대
✅ 엑스포타워

장점

  • 코스 완성도 높음
  • "이달의" 특별함 표현
  • 동선 자연스러움

단점

  • 반경 내 코스 없으면 빈 화면
  • 구현 복잡도 높음
C

코스 선택 없앰 — Gemini가 전부 통합

코스 서브장소 이름을 Gemini 프롬프트 힌트로만 전달. 사용자에게는 코스 선택 단계 없음. Gemini가 축제 + 힌트 장소 + 근처 장소를 통합해서 최적 코스 생성.

사용자가 보는 것: 축제 선택 → 장소 목록 → 생성
백그라운드: 코스 힌트가 Gemini 프롬프트에 포함됨
→ 사용자 경험 단순화

장점

  • UX 가장 단순
  • 축제 강조 자연스러움
  • 코스 충돌 문제 없음

단점

  • Gemini 신뢰도 의존
  • 관광공사 코스 활용 안 보임
  • 공모전 어필 약함
💡 내 추천: A + B 하이브리드 — 축제는 항상 고정(A), 코스 장소는 지리적 근접성 필터로 자동 선택(B). 코스 선택 UI는 없애고 백그라운드에서 자동 처리. 사용자는 단순하게, 데이터는 풍부하게.
