데이터 활용 전략

TourAPI + 카카오 + Gemini 조합으로 공모전 데이터 활용도 극대화

📡 새 플로우 × 데이터 매핑
1
홈 — 이달의 추천 여행지
TourAPI 여행코스(25) searchFestival2 큐레이션 하드코딩
관광공사 검증된 여행코스 데이터 + 이번 달 인기 축제로 추천 카드 구성
↓
2
날짜 + 지역 + 출발지 입력
카카오 키워드검색 (출발지) 카카오 경로API (이동시간)
출발지 → 목적지 이동시간 계산. 당일치기면 왕복 제외한 실제 관광 가능 시간 산출
↓
3
장소 선택 — AI 추천 + 사용자 조정
searchFestival2 (행사) locationBasedList2 (관광지12, 문화시설14, 레포츠28) searchStay2 (숙박32, 1박이상) 카카오 맛집검색 (보완) Gemini 추천 선택
TourAPI 음식점(39)은 데이터 부실 → 카카오 키워드로 맛집 보완. Gemini가 실제 관광 가능 시간 기준으로 장소 수 자동 결정
↓
4
코스 결과
detailIntro2 (영업시간, 요금) detailImage2 (이미지) 카카오맵 내비 카카오톡 공유
각 장소 영업시간 실제 반영. 이미지 풍부하게. 장소별 카카오 내비 + 전체 공유
현재 사용중
🎪
행사/축제
searchFestival2 (type:15)
날짜 기반 축제 검색. 지역코드 조합 가능
→ 핵심 유지
현재 사용중
🏔️
관광지
locationBasedList2 (type:12)
위치 반경 기반 관광지 목록. 최대 20km
→ 핵심 유지
신규 추가
🗺️
공사 검증 여행코스
areaBasedList2 (type:25)
관광공사가 큐레이션한 지역별 여행코스. 이달의 추천에 활용
→ 홈 큐레이션 베이스
신규 추가
🏛️
문화시설
locationBasedList2 (type:14)
박물관, 미술관, 전시관. 지금 코스에서 빠져있음
→ 장소 선택에 추가
신규 추가
🏄
레포츠
locationBasedList2 (type:28)
액티비티, 스포츠 시설. 체험형 여행에 필수
→ 액티비티 옵션
신규 추가
🏨
숙박
searchStay2 (type:32)
1박 이상 선택 시 숙소 위치 기반 동선 최적화
→ 1박2일 이상에서 활용
신규 추가
⏰
영업시간/요금
detailIntro2
실제 운영 시간 기반 코스 배치. "문 닫은 곳" 포함 방지
→ 코스 정확도 향상
카카오 보완
🍽️
맛집 (카카오)
카카오 키워드 검색 API
TourAPI 음식점(39) 데이터 부실 → 카카오 맛집 검색으로 보완. 리뷰 수 기반
→ 음식점 대체/보완
💡 공모전 심사 포인트
📊
TourAPI 5개 이상 데이터 타입 활용 → 데이터 활용도 점수 높음 (현재는 2개)
🗺️
카카오 API는 경로 탐색 + 내비 + 공유 3가지 모두 활용 → 카카오 연동 완성도
🤖
Gemini가 이동시간 기반 코스 최적화 → 단순 추천이 아닌 실용적 AI 활용 어필 가능
