## 배경

축제 카드 3가지 문제:
1. 지역명이 너무 작고 색상이 이미지 위에서 안 보임
2. 상태 뱃지(진행중/예정/종료) 시각적 구분이 약함
3. 정렬이 없어서 종료된 축제가 위에 노출됨

---

## Fix 1 — `src/components/FestivalCard.tsx`

### 1-A. 지역명 수정

기존:
```tsx
<p className="text-secondary font-bold text-[10px] mb-1">{region}</p>
```

교체:
```tsx
<p className="text-white/90 font-bold text-xs mb-1">{region}</p>
```

### 1-B. 상태 뱃지 수정

기존:
```tsx
<div className={`absolute top-3 right-3 px-2 py-1 bg-white/90 backdrop-blur rounded text-[10px] font-bold shadow-sm z-10 ${
  status === '진행중' ? 'text-primary' : status === '예정' ? 'text-secondary' : 'text-slate-400'
}`}>
  {status}
</div>
```

교체 (배경색으로 차별화 + 종료 카드 어둡게):
```tsx
<div className={`absolute top-3 right-3 px-2 py-1 rounded text-[10px] font-bold shadow-sm z-10 text-white ${
  status === '진행중' ? 'bg-green-500' :
  status === '예정' ? 'bg-primary-container' :
  'bg-slate-400'
}`}>
  {status}
</div>
```

종료 카드 전체 어둡게 처리 — 최상위 div에 조건부 클래스 추가:
```tsx
<div 
  onClick={onClick}
  className={`group cursor-pointer relative bg-surface-container-high rounded-xl overflow-hidden hover:shadow-2xl transition-all duration-500 aspect-[3/4] ${
    status === '종료' ? 'opacity-50' : ''
  }`}
>
```

## Fix 2 — 정렬 로직 추가

`getFestivalStatus`가 이미 utils.ts에 있음. 이걸 활용해서 정렬.

정렬 순서: 진행중(0) → 예정(1) → 종료(2)

### 2-A. `src/pages/HomePage.tsx`

`useEffect` 안 `setFestivals(data)` 부분 수정:

```tsx
import { getFestivalStatus } from '../lib/utils';
```
(이미 import 안 되어 있으면 추가)

```tsx
const sorted = [...data].sort((a, b) => {
  const order = { '진행중': 0, '예정': 1, '종료': 2 };
  const sa = getFestivalStatus(a.eventstartdate, a.eventenddate) as keyof typeof order;
  const sb = getFestivalStatus(b.eventstartdate, b.eventenddate) as keyof typeof order;
  return (order[sa] ?? 3) - (order[sb] ?? 3);
});
setFestivals(sorted);
```

### 2-B. `src/pages/FestivalsPage.tsx`

동일하게 적용. getFestivalStatus import 추가 후 동일한 정렬 코드.
