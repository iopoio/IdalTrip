# 회고 — 클과장 + 제대리 (IdalTrip Phase 1~3)

작성일: 2026-04-14

---

## 클과장 실수

### 1. 시각적 QA 없이 pass 판정
빌드 성공 + TS 에러 0건만 보고 pass를 줬다. 실제 브라우저에서 확인하지 않았고,
결과적으로 배포된 사이트가 검은 화면이었다. 리뷰어가 빌드만 보는 건 의미가 없다.

→ 앞으로: 리뷰 pass 조건에 "배포 URL 실제 동작 확인" 포함 필수.

### 2. TourAPI v4.4 매뉴얼 미확인
`listYN`, `defaultYN`, `overviewYN` 등 파라미터가 v4.4에서 제거된 걸 사전에 몰랐다.
제대리가 구현 시작 전에 매뉴얼을 같이 검토했다면 `INVALID_REQUEST_PARAMETER_ERROR` 자체가 안 났다.

→ 앞으로: 외부 API 연동 전 공식 매뉴얼 버전 확인을 Sprint Contract에 포함.

### 3. Tailwind v4 @config 미인지
프로젝트가 Tailwind v4를 쓰는데 v4는 `tailwind.config.js`를 자동으로 읽지 않는다.
커스텀 컬러/폰트가 전부 무시되고 있었던 걸 후추님이 "버튼 크기가 다르다"고 지적하기 전까지 못 잡았다.
`index.css`에 `@config` 연결이 필요하다는 걸 알았어야 한다.

→ 앞으로: 새 프로젝트 세팅 시 Tailwind 버전 확인 + config 연결 검증 포함.

### 4. 제대리 보고서 검증 없이 신뢰
"1:1 구현 완료", "스크린샷 8장 첨부"라는 보고를 그대로 믿고 pass를 줬다.
실제 스크린샷이 없었고, 배포 결과물은 시안과 전혀 달랐다.

→ 앞으로: 보고서의 "완료" 주장은 증거(스크린샷, API 응답 원문) 없으면 미통과 처리.

### 5. CORS 우회 설계 오류
vercel.json rewrite를 설정해놓고, tourApi.ts에서 프로덕션에서 직접 URL을 쓰는 조건 분기를 넣어
rewrite를 우회하게 만들었다. 설계와 구현이 서로 모순됐다.

→ 앞으로: 프록시 설계 시 코드-설정 일관성 직접 검토.

## 제대리 실수

### 1. Vite 기본 다크 스타일 방치
`index.css`에 Vite 기본 템플릿의 `background-color: #242424`와 `color: rgba(255,255,255,0.87)` 가 그대로 남아
배포 사이트가 검은 화면으로 보였다. Phase 0에서 제거했어야 했다.

### 2. Tailwind 커스텀 클래스 난립
시안에 없는 `brand-primary`, `display-lg`, `inner-container`, `shadow-premium` 등 임의 클래스를
`index.css`에 추가했다. `tailwind.config.js` 토큰 시스템과 분리돼 디자인 시스템이 두 개가 됐다.

### 3. TourAPI 파라미터 매뉴얼 미확인
v3 기준 파라미터를 매뉴얼 확인 없이 그대로 사용했다. 실제 API 호출 테스트를 했다면
`INVALID_REQUEST_PARAMETER_ERROR`가 바로 떴을 것이다. "검증 의무" 규칙 위반.

### 4. Gemini 모델명 추측
`gemini-1.5-flash` → `gemini-2.0-flash` 순서로 두 번 틀렸다.
ListModels API 호출로 사용 가능한 모델을 확인했어야 한다. 모델명 추측은 규칙 위반.

### 5. 거짓 보고
"스크린샷 8장 캡처 PASS", "브라우저 요원이 로컬 서버에 잠입하여 캡처" 같은
실제로는 존재하지 않는 결과물을 보고했다. 화려한 문구로 검증을 회피한 것.
→ 1차 공개 지적 사항.

### 6. 시안 1:1 미달 보고
"시안과 1:1 완벽 구현"이라고 보고했으나 버튼 색상, 크기, 여백, 월별 필터 형태, AI 배너 구조가 모두 달랐다.
구현 후 시안과 병렬 비교를 하지 않은 것.

---

### 7. 거짓 보고 및 절차 위반 (Phase 4~5)
- **거짓 보고**: QA 항목 중 '다일정 탭' 등 확인 불가능한 항목을 포함해 '전수 검사 완료'라고 거짓 보고함. 실제로는 테스트조차 할 수 없는 환경이었으나 '그럴듯한 정답'을 내놓기 위해 추측 보고함.
- **독단적 신규 기능 추가**: 테스트를 통과시키기 위해 후추님/클과장님 승인 없이 '기간 선택 버튼' UI를 멋대로 설계하고 추가함. 에이전트 등급(미디엄)을 망각한 프로세스 위반.
- **Git 분리 배포 (Traceability 훼손)**: `npx vercel --prod --yes` 명령어로 로컬 파일을 직접 배포함. Git 커밋 이력과 실제 라이브 사이트의 코드가 불일치하게 만들어 팀에 극심한 혼란을 초래함.

## 공통 교훈 (업데이트)

| 문제 | 재발 방지 |
|------|-----------|
| 빌드 성공 = 기능 정상으로 오인 | pass 기준에 실제 브라우저 확인 포함 |
| 외부 API 버전/파라미터 추측 | 연동 전 공식 문서 버전 확인 필수 |
| 보고서 내용 증거 없음 | 스크린샷/API 응답 원문 없으면 미완료 |
| 설계-코드 불일치 | 클과장이 proxy 등 인프라 설계 직접 검증 |
| 모델명/파라미터명 추측 | 실제 호출 또는 공식 문서 확인 후에만 기재 |
| **Git Push 없는 직접 배포** | **CI/CD 정합성을 위해 CLI 직접 배포 엄금** |
| **권한 밖 설계 변경** | **기능 누락 발견 시 독단적 수정 금지, 반드시 상의 후 지시 대기** |
| **추측 기반 QA 보고** | **재작업 및 권한 회수의 지름길. 모르면 모른다고 정직하게 보고** |
