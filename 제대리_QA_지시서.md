## QA 체크리스트

### [ 홈 ]
```
□ 히어로 이미지 + 텍스트 오버레이 표시
□ 지역 선택 (전체 포함 7개 버튼)
□ 날짜 선택 작동
□ 현재 진행중인 축제 카드 표시 + 상태 뱃지 (진행중/예정/종료)
□ 탐색하기 버튼 → 탐색결과 페이지 이동
□ 로고 표시 (탑바 + 하단 푸터)
```

### [ 탭바 ]
```
□ 홈 / 축제소식 / 최근코스 3탭 표시
□ 각 탭 클릭 시 페이지 이동 정상 작동
□ 현재 페이지 탭 활성화 표시
```

### [ 축제소식 ]
```
□ 월 필터 (현재 달부터 12월까지 표시)
□ 지역 필터 7개 (전체/서울경기/강원/충청/전라/경상/제주)
□ 진행중 → 예정 → 종료 순 정렬
□ 데이터 없을 때 empty state 표시
```

### [ 최근코스 ]
```
□ 코스 생성 후: 카드 + 코스 다시 보기 버튼 표시
□ 코스 없을 때: 안내 문구 + 탐색 시작하기 버튼 표시
```

### [ 탐색결과 ]
```
□ 지역/날짜 기준 장소 로드
□ 출발지 입력 (탭 위 상단 위치)
□ 자가용/대중교통 토글 작동
□ 전체/축제/관광지/맛집 탭 필터
□ 카드 선택/해제 (오렌지 테두리 + 체크)
□ 탭별 데이터 없을 때 empty state (🎪/🏞️/🍽️/🗺️ + 문구)
□ 하단 AI 코스 짜기 버튼 표시 (탭바 위에 보임)
□ 한국관광공사 TourAPI 출처 문구 표시
```

### [ 코스결과 ]
```
□ 지도 상단 표시 + 번호 마커 + 폴리라인
□ 출발지 블록 표시 (📍출발지명 + 첫 장소까지 이동시간)
□ 세로 타임라인 (번호마커 → 장소카드 → 이동시간 반복)
□ 여행 브리핑 / 예상비용 섹션 표시
□ 다일정(2일 이상) 시 1일차/2일차 탭 표시 및 전환
□ 카카오내비/길찾기 버튼 클릭 시 이동
□ 카카오톡 공유 버튼 클릭 시 공유 시트 열림
□ 공유 메시지: 장소 3개 + 안내문구 포함 확인
□ 코스결과 페이지 새로고침 시 404 없음
□ 코스 생성 후 최근코스 탭에 저장됨
```

### [ 공통 ]
```
□ 모든 페이지 새로고침 시 404 없음
□ 뒤로가기 정상 작동
```

## 보고 형식

각 항목 ✅/❌ 표시. ❌ 항목은 증상 한 줄 추가.  
빠진 항목 있으면 보고로 인정 안 함.

### 증거 첨부 규칙 (필수)
- ✅ 표시 시, "어떻게 확인했는지" 관찰 내용 1줄 필수.  
  예: `✅ 출발지 블록 표시 — 서울역 입력 후 코스 생성, 타임라인 최상단 📍서울역 + 🚗 40분 표시 확인`
- 관찰 내용 없는 ✅는 불인정.
- 실제로 확인하지 못한 항목은 ✅ 금지 → [미확인] 태그 후 이유 기술.
- 전체 항목이 ✅인 보고서는 의심한다. ❌가 0개인 경우 재확인 근거 별도 기술 필수.
