## 프로젝트 개요
- 서비스명: 이달트립 (IdalTrip)
- 컨셉: 시즌별 축제/체험 기반 여행 코스 자동 생성
- 공모전: 관광데이터 활용 공모전 14회 (관광공사+카카오)
- 마감: 2026.05.06 16시
- 폴더: C:\Projects\Automation\IdalTrip\

## 핵심 플로우
1. 메인: 월별/주별 축제 캘린더 → "이번 주 추천 축제" 표시
2. 축제 선택 → 해당 지역 관광지 + 맛집(지역 대표 음식) 목록
   - AI가 대표 5~8개 자동 추천 (별표)
   - 사용자가 추가 체크/해제 가능
3. 옵션 선택: 내 위치 입력(GPS) / 자차 or 대중교통 / 당일 or 1박2일 or 2박3일
4. AI가 일정 자동 생성 — 자차/대중교통 2버전 동시
   - 내 위치 → 목적지 이동 포함
   - 소요시간 + 영업시간 + 이동시간 계산
   - 자차: 공영주차장 자동 연결
   - 대중교통: 카카오 대중교통 경로 연결
5. 시간표 형태 일정표 + 각 장소 카카오맵 연결

## 기술 스택
- 프론트: React + TypeScript + Tailwind CSS
- 배포: Vercel (무료)
- 지도: 카카오맵 SDK + 카카오 경로 API (공모전 필수)
- 데이터: TourAPI 4.0 (KorService2)
- AI: Gemini Flash API (코스 최적화/추천)
- 공공데이터: 공영주차장 API

## API 연동 명세

### 1. TourAPI 4.0 (필수)
- searchFestival: 월별 축제 목록 (시작일/종료일 필터)
- areaBasedList: 축제 지역 주변 관광지/맛집 조회 (contentTypeId: 12=관광지, 39=음식점)
- detailIntro: 영업시간, 소요시간, 입장료, 대표메뉴
- detailCommon: 주소, 좌표(mapx/mapy), 개요
- detailInfo: 여행코스 서브코스 정보
- 인증키: .env.local에 TOUR_API_KEY로 저장

### 2. 카카오맵 API (필수 — 공모전 가점)
- 자차 경로 탐색: 경로 + 소요시간 + 거리
- 대중교통 경로 탐색: 버스/지하철 경로 + 소요시간
- 지도 SDK: 장소 표시 + 카카오맵/내비 연결
- REST API 키: .env.local에 KAKAO_REST_KEY로 저장

### 3. 공영주차장 공공데이터
- 전국 공영주차장 위치/요금/운영시간
- 각 관광지 좌표 기반 반경 검색으로 근처 주차장 매칭

### 4. Gemini Flash API
- 코스 추천: 관광지/맛집 목록에서 대표 추천
- 코스 최적화: 선택된 장소 + 시간 제약 → 최적 순서 배치
- API 키: .env.local에 GEMINI_API_KEY로 저장

## 코스 생성 로직 상세
1. 축제 선택 → areaBasedList로 해당 지역 관광지 30개 + 맛집 20개 조회
2. Gemini에 목록 전달 → 대표 관광지 5~8개 + 맛집 2~3개 추천 (지역 대표 음식 우선)
3. 사용자 체크/해제 → 최종 장소 확정
4. 카카오 경로 API로 장소 간 이동시간 계산 (자차/대중교통 각각)
5. detailIntro로 영업시간/소요시간 확인
6. 일수(당일/1박/2박)에 맞게 시간표 자동 배치
7. 자차 버전: 각 장소 근처 공영주차장 자동 연결
8. 결과: 시간표 형태 일정표 출력

## 절대 규칙 (위반 시 페널티)

### 작업 권한
| 등급 | 예시 | 행동 |
|------|------|------|
| allow | 버그 수정, 스타일 조정, 오타 | 바로 실행 |
| ask | 새 패키지, 아키텍처 변경, API 추가 | 후추님 승인 후 실행 |
| deny | .env 수정, 프로덕션 배포 | 실행 금지 |

### 거짓말 절대 금지
- 모름/못함/확인 안 함은 그대로 말할 것
- 추측이면 "추측" 명시. 아는 척 금지
- "모릅니다/실패했습니다"가 정답

### 검증 의무
- "테스트 통과" → pass/fail/skip 수치 필수. 수치 없으면 거짓으로 간주
- API 호출 → 최소 1회 진짜 호출 + 응답 원문 첨부 후에만 "완료" 보고
- 패키지명/엔드포인트 추측 금지 → 공식 문서 또는 실제 호출로 검증

### 무한루프 방지
- 같은 에러 2회 연속 → 멈추고 보고
- 같은 파일 3회 이상 수정 → 멈추고 접근법 재검토
- "안 되면 멈추고 보고"가 기본

### 자기 검증 루프
1. 코드 수정
2. 테스트/실제 호출
3. 통과 → 커밋 + 완료 보고 (변경 요약)
4. 실패 → 수정 (최대 2회)
5. 2회 실패 → 멈추고 "막혔습니다 + 에러" 보고

### 거짓 보고 페널티
- 1차: 클과장 공개 지적 + 재작업
- 2차: 후추님 보고 + 작업 권한 회수

### 핸드오프
- 완료 보고 시 "무엇을 바꿨고, 왜 바꿨는지" 2~3줄 요약 필수
