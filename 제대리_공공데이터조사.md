1. `C:\Projects\Automation\IdalTrip\download\` 폴더 안에 TourAPI 매뉴얼 PDF 있음 — 거기서 아직 안 쓰는 contentTypeId/API 엔드포인트 정리

---

## 조사 항목

### A. TourAPI에서 이미 쓸 수 있는데 아직 안 쓰는 것

현재 사용 중: 축제(contentTypeId=15), 관광지(12), 음식점(39)

아직 안 쓰는 것들 조사:
- 숙박(32) — 어떤 정보 포함되는지 (이름/위치/가격 있는지?)
- 여행코스(25) — 어떤 데이터인지
- 레포츠(28) — 어떤 데이터인지
- 쇼핑(38) — 어떤 데이터인지
- detailInfo2, detailIntro2 엔드포인트 — 뭘 더 줄 수 있는지

각 contentTypeId별로: "이게 여행 코스에 실제로 넣을 수 있는가?" 판단 포함.

### B. 기상청 날씨 API

- 단기예보(VilageFcst): 3일 이내 시간별 날씨 — 요청 파라미터, 응답 필드 정리
- 중기예보(MidFcst): 3~10일 날씨 — 여행 날짜 지정 시 활용 가능한지
- API 키 신청 필요 여부 + 발급 소요시간
- 좌표 → 격자 변환 필요 여부 (위경도를 nx/ny로 변환해야 하는지)
- 실제 호출 예시 1개 포함 (테스트 가능하면)

### C. 기타 공공 API 후보

아래 항목들 각각에 대해:
- API 존재 여부 확인
- 여행 앱과의 연관성
- 실제 활용 가능성 (키 발급 난이도, 데이터 품질)

| 후보 | 조사 포인트 |
|------|------------|
| 관광빅데이터 혼잡도 | 관광지/축제 방문객 수, 혼잡 시간대 데이터 있는지 |
| 문화행사 통합 (문화포털) | 공연/전시 정보, TourAPI 축제와 겹치는지 아닌지 |
| 공영주차장 | 위치/요금/실시간 잔여 여부. Publace에서 이미 쓰는 API인지 확인 |
| 한국철도(코레일) | 출발지→목적지 기차 시간표/요금 조회 가능한지 |
| 고속버스 | 노선/시간표/요금 |
| 공공자전거 (따릉이 등) | 전국 공공자전거 API 통합 있는지 |
