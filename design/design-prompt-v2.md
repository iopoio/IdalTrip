# 디자인 프롬프트 v2 — 스티치 재요청용

아래 전체를 그대로 복사해서 구글 AI 디자인 툴에 입력.

---

**서비스명:** 이달의 여행 (IdalTrip)

**한줄 설명:** "이번 달 어디 갈까?" — 현재 달의 축제를 중심으로 AI가 맞춤 여행 코스를 자동 생성하고, 카카오맵으로 바로 길찾기까지 연결해주는 국내 여행 플래너.

**타겟:** 20~30대 국내 여행자. 모바일에서 빠르게 여행 계획을 세우고 싶은 사람.

**플랫폼:** 모바일 퍼스트 반응형 웹 (React + Tailwind CSS). 아이콘은 lucide-react 세트 사용.

---

**디자인 원칙 (반드시 지켜주세요):**

1. 컬러 시스템을 모든 페이지에서 동일하게 사용할 것. 페이지마다 다른 색상 토큰을 쓰지 말 것.
2. UI 텍스트는 전부 한국어로. 영어는 브랜드명 "IdalTrip"만 허용. 네비게이션, 버튼, 라벨 모두 한국어.
3. 타이포그래피 위계를 명확히: 페이지 제목(bold) > 섹션 제목(semibold) > 본문(regular) > 캡션(light). 모든 텍스트를 extrabold로 쓰지 말 것.
4. "이번 달"이라는 시즌감이 첫 화면에서 바로 느껴져야 함. 현재 월이 크게 강조되고, 그 달의 대표 축제가 즉시 보여야 함. 단순한 pill 탭이 아니라 시각적으로 "지금 5월" 같은 시간적 맥락이 드러나는 디자인.
5. 모바일 하단 영역 정리: BottomNav와 고정 액션 버튼이 겹치지 않도록. 한 화면에 하단 고정 요소는 하나만.
6. 과도한 그림자와 라운딩 자제. 카드 라운딩은 최대 1.5rem, 그림자는 은은하게.

---

**필요한 화면 (4개):**

**화면 1: 메인 홈**
- 상단 네비: 로고 "IdalTrip" + 검색 + 프로필
- 히어로: "5월의 여행" 같은 시즌 헤드라인 + 이번 달 대표 축제 1개가 배경 이미지로. 축제명과 기간이 바로 보임.
- 월별 필터: 1~12월 가로 스크롤. 현재 월이 시각적으로 확실히 구분됨 (색상 + 크기가 아니라 "5월"이라는 텍스트와 축제 수가 함께 표시되는 식).
- 축제 카드 그리드: 사진, 축제명, 지역, 기간. 카드 클릭 시 상세로 이동. 이미지가 없는 경우의 placeholder도 디자인해줄 것.
- AI 추천 배너: "내 취향에 맞는 축제 찾기" — 여행/지도 관련 아이콘 사용 (로켓 말고 나침반, 지도핀, 경로 등)
- 모바일 하단 네비: 홈 / 내 여행 / 축제 (한국어)

**화면 2: 축제 상세 + 옵션 선택**
- 상단: 축제 대표 이미지 + 축제명 + 지역 + 기간
- AI 추천 명소/맛집 목록: 카드형, 체크박스로 선택/해제 가능. 카테고리 구분 (명소/맛집 라벨)
- 옵션 영역: 출발지 입력 / 이동수단 (자차 or 대중교통) / 일정 (당일 or 1박2일 or 2박3일)
- 하단 고정: "AI 코스 생성하기" 버튼 (이 화면에서는 BottomNav 숨김)

**화면 3: AI 로딩 → 코스 결과**
- 로딩 상태: AI가 코스를 만들고 있다는 애니메이션 + 스켈레톤 프리뷰
- 완료 후 결과:
  - AI 추천 사유 말풍선 (왜 이렇게 구성했는지 한줄 설명)
  - 코스 요약 카드: 총 소요시간, 예상 비용, 장소 수
  - 타임라인 일정표: 시간 / 장소명 / 카테고리 아이콘 / 소요시간 / 간단 메모. 각 장소에 "카카오맵으로 보기" 버튼.
  - 자차 / 대중교통 탭 전환
  - 1박2일 이상이면 Day1 / Day2 구분
  - 하단: "이 코스 저장하기" + "다른 코스 추천받기" 버튼

**화면 4: 코스 결과 — 지도 뷰**
- 상단: 카카오맵 영역 (전체 경로가 지도 위에 표시, 각 장소에 번호 마커)
- 하단: 장소 리스트 (카드 형태, 가로 스크롤). 카드 탭 시 지도에서 해당 마커로 이동.
- 각 장소 카드: 장소명 + 카테고리 + 이동시간 (이전 장소로부터) + "카카오맵 네비" 버튼
- 자차일 경우 주차장 정보 표시

---

**톤앤무드:** 여행의 설렘이 느껴지되 과하지 않은 모던함. 사진이 주인공. 깔끔하고 신뢰감 있는 느낌. 축제의 활기는 컬러 포인트로만 표현하고, 전체 레이아웃은 정돈되어 있을 것.

**참고:** 이전 시안에서 오렌지 + 블루 조합은 좋았음. 유지해도 됨. 단, 4개 화면 모두 동일한 컬러 토큰을 사용할 것.
