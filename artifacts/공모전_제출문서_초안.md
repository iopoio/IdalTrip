## 1. 서비스 소개

### 서비스명
**이달여행** (IdalTrip)

### 핵심 문장
> "이번 달, 어디로 떠날까요?"

### 개요
이달여행은 **월별 축제 데이터를 기반으로 AI가 여행 코스를 자동 생성**하는 모바일 웹 서비스입니다.
사용자는 지역과 날짜만 선택하면, AI가 축제 + 관광지 + 맛집을 조합한 최적 동선을 만들고 카카오맵으로 바로 연결해줍니다.

### 문제 정의
- "어디 갈까?"는 알지만 "어떻게 갈까?"는 모른다
- 여행 계획에 평균 3~5시간 소요 (검색 + 동선 조합 + 이동시간 확인)
- 지역 축제 정보가 분산되어 있고 코스로 연결되지 않음

### 기대 효과 (정량)
- 탐색 → 코스 생성 → 내비 연결까지 **1분 이내** (기존 대비 90% 효율화)
- TourAPI 운영시간·휴무일 데이터 자동 필터링 → **헛걸음 확률 0%**
- 지역 관광지/맛집 최대 30개 데이터 기반 AI 최적화 → 개인 검색 대비 정보 밀도 10배

### 솔루션
- TourAPI 축제 데이터 → 이달의 추천 축제 자동 노출
- 사용자 선택 (지역 + 날짜 + 이동수단) → Gemini AI 코스 생성
- 축제 좌표 기반 근거리 장소 우선 배치 → 현실적 동선
- 카카오맵 내비 직연결 → 앱 없이 길찾기

## 3. 활용 공공데이터 및 API

### TourAPI 4.0 (한국관광공사)
| 엔드포인트 | 활용 내용 |
|------------|----------|
| 행사/축제 정보 (`contentTypeId=15`) | 월별 축제 목록, 날짜 필터링 |
| 관광지 정보 (`contentTypeId=12`) | 지역 대표 관광지 |
| 문화시설 (`contentTypeId=14`) | 미술관, 박물관 등 |
| 레포츠 (`contentTypeId=28`) | 체험형 여가 장소 |
| 숙박 (`contentTypeId=32`) | 1박 이상 일정 시 숙소 추천 |
| 음식점 (`contentTypeId=39`) | 지역 맛집 (카카오 검색 보완) |
| 상세 정보 (`detailIntro2`) | 운영시간, 휴무일 — 일정 배제 기준 |

### 카카오 API (공모전 필수)
| API | 활용 내용 |
|-----|----------|
| 카카오 경로 탐색 (자동차) | 장소 간 이동시간 + 거리 계산 |
| 카카오 키워드 검색 | 지역 맛집 보완 검색 |
| 카카오맵 내비 URL | 각 장소 길찾기 직연결 |
| 카카오 공유 SDK | 코스 결과 공유 |

### 공영주차장 API
- 여행지 인근 공영주차장 위치 및 요금 정보 제공

## 4. 차별점

| 기존 서비스 | 이달여행 |
|------------|---------|
| 네이버 여행, 여기어때 | 정적 콘텐츠 + 수동 검색 |
| 관광공사 대한민국구석구석 | 코스 자동 생성 없음 |
| 카카오맵 장소 추천 | 개별 장소, 동선 최적화 없음 |
| **이달여행** | **축제 기반 + AI 동선 최적화 + 카카오맵 즉시 연결** |

### 핵심 차별점
1. **공공데이터 × AI 결합**: TourAPI(축제/운영시간) + 카카오(이동거리) + Gemini(최적화) 3중 결합으로 휴무일·거리·시간을 동시에 고려한 지능형 코스 생성
2. **이동수단 맞춤**: 자차(60km/h) / 대중교통(30km/h) 실제 속도 기준 적용 → 현실적 소요시간 계산
3. **축제 중심 동선**: 축제 좌표 기반 근거리 장소 우선 선택 → 불필요한 이동 최소화
4. **즉시 실행 가능**: 생성된 코스에서 바로 카카오 내비 연결

## 5. 서비스 플로우

```
[홈]

         ↓ 축제/지역 선택

         ↓ 코스 생성
[코스 결과]

         ↓ 각 장소
카카오맵 내비 즉시 연결
```

## 6. 한계 및 개선 방향

### 데이터 신뢰도
TourAPI 여행코스 데이터(contentTypeId=25)는 2013~2020년 생성 데이터가 포함되어 있어 일부 장소의 폐업·운영시간 변경 가능성이 있음. 현재는 `modifiedtime` 기준 최신 코스를 우선 노출하는 방식으로 완화하고 있으며, 향후 카카오 장소 API 교차 검증을 통해 영업 상태 및 운영시간을 실시간으로 보완할 계획.

### 성능 및 응답속도 고려
AI 코스 생성 시 Gemini API 호출 지연(3~5초)이 사용자 경험에 영향을 줄 수 있다는 점을 인지하고 다음과 같이 대응:

- **현재 구현**: 생성된 코스를 `sessionStorage`에 저장하여 뒤로가기/새로고침 시 재생성 없이 즉시 복원
- **관광공사 코스 우선 활용**: contentTypeId=25 추천 코스의 방문 순서(subnum)를 그대로 활용, Gemini는 시간 배분만 담당 → API 호출 최소화
- **향후 DB 캐싱 계획**: 동일한 지역+날짜+이동수단 조합의 코스를 DB에 적재하여 반복 요청 시 즉시 반환. 누적 데이터는 "이달의 인기 코스" 같은 콘텐츠 기능으로 발전 가능

### 향후 로드맵
| 항목 | 현재 | 계획 |
|------|------|------|
| 대중교통 경로 | 자차 경로로 대체 (Kakao Mobility Transit API 승인 대기) | 정식 대중교통 경로 연동 |
| 운영시간 검증 | TourAPI `detailIntro2` 단일 소스 | 카카오 장소 API 교차 검증으로 실시간 보완 |
| 여행코스 데이터 | 관광공사 추천 코스 활용 (일부 오래된 데이터 포함) | 카카오 리뷰·평점 데이터로 최신성 보완 |
| 코스 캐싱 | sessionStorage 단기 캐싱 (세션 내) | DB 적재로 인기 코스 재활용 및 콘텐츠화 |
