# 기술 설명서 및 아키텍처 초안

> **목적:** 공모전 기술 부문 심사 및 GitHub README 첨부용 문서

## 1. System Architecture
- **Frontend:** React, TypeScript, Tailwind CSS, Vite
- **Hosting / CI·CD:** Vercel (Edge Network 활용)
- **Map & Routing:** Leaflet (OpenStreetMap) + Haversine Formula + Kakao Map Link (내비 연결)
- **AI Engine:** Google Gemini Flash API

## 2. API & Data Flow
### TourAPI 4.0 매시업 구조
1. **행사정보조회 (`searchFestival2`):** 사용자가 선택한 지역/날짜 기반 축제 조회.
2. **키워드검색 (`searchKeyword2`):** 축제 인근의 관광지(12) 및 음식점(39) 목록 추천.
3. **공통정보조회 (`detailCommon1`):** 선택한 장소의 상세 개요 및 메인 이미지(originImgurl) 로드.

### AI 코스 생성 플로우 (Gemini)
1. **Input:** 사용자가 선택한 장소(Place) 데이터 배열 및 여행 기간.
2. **Processing:**
   - Gemini 프롬프트를 통해 장소들의 특성(축제/식사/관람) 분석.
   - 하버사인(Haversine) 공식을 자체 구현하여 장소 간 직선거리 및 예상 이동시간(30km/h 기준) 도출 — 경량 연산으로 Network Latency 없이 코스 생성 즉시 렌더링.
3. **Output:** `day`, `time`, `stay_duration`, `move_time`이 포함된 최적화된 JSON 타임라인 반환.

## 3. Technical Challenges & Solutions
**Issue:** Kakao Local / Mobility API 의존 시 403 에러 및 트래픽/라이선스 제약 발생 (Vercel 환경).
**Solution:** 
- **공개 API 생태계 재설계 (Open Ecosystem Redesign):** 지도 렌더링을 Kakao Map SDK에서 OpenStreetMap(Leaflet)으로 전면 교체하여 토큰 제한 이슈를 영구적으로 해결하고 유연한 확장이 가능하도록 아키텍처 개선.
- **경량 연산으로 즉시 응답성 확보:** 장소 간 이동 시간 계산을 자체 하버사인(Haversine) 공식 기반 연산 로직으로 전환. 외부 라우팅 API 호출 딜레이(Network Latency)를 없애 코스 생성 즉시 결과를 노출함. 향후 정밀한 경로 탐색을 위한 OSRM 연동 로드맵 구축.
- **Seamless UX:** 최종 사용자 길안내는 카카오맵 공개 URL Scheme (`https://map.kakao.com/link/map/...`)을 활용하여 API 키 없이도 심리스한 네비게이션 경험 유지.

## 4. UI/UX 기술 적용
- **가로 스와이프 - 지도 연동:** `IntersectionObserver`를 활용하여 하단 카드가 스크롤될 때마다 상단 지도의 마커 포커스가 부드럽게 동기화되도록 구현.
- **반응형 설계:** 모바일 퍼스트 뷰(`100svh`, 하단 Safe Area Inset)를 적용하여 앱과 동일한 네이티브 경험 제공.
