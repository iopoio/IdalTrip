# 이달여행(IdalTrip) — 양식1 제안서 초안 v2

> **목적**: 공식 양식1 hwp에 복붙하기 위한 4개 섹션 원고

> **컨셉 (v2 갱신)**: 바쁘고 귀찮은 현대인을 위해 — 고민할 필요 없이 이번 달 갈 곳을 알려준다
> **차별화 표현 원칙**: 경쟁 서비스 직접 비교 X. 사용자 가치 중심으로 은근한 차별화.

## 1) 서비스 기획배경 및 필요성

### 1. 서비스 기획 배경

여행을 떠나기 전, 가장 먼저 마주치는 건 풍경이 아니라 **결정 피로**다. 어디로 갈지, 언제가 적기인지, 무엇을 할지 — 평균적으로 사람들은 주말 여행 하나를 정하는 데 3~5시간을 검색에 쓴다. 네이버에서 축제를 찾고, 지도에서 관광지를 찍어보고, 블로그에서 맛집을 검증하고, 카카오맵에서 동선을 그린다. 이 과정에서 정작 여행의 즐거움보다 **계획의 부담**이 더 크게 다가온다.

한국관광공사가 축제·관광지·음식점·숙박 데이터를 OpenAPI로 공개하고 있지만, 일반 사용자가 이를 직접 활용하긴 어렵다. 데이터는 풍부한데 **"이번 달 나에게 맞는 여행"** 이라는 맥락으로 묶이지 않기 때문이다.

**이달여행**은 이 간극을 좁힌다. 사용자가 입력하고 검색하고 비교할 필요 없이, 진입 즉시 **"이번 달, 지금 갈 만한 곳"** 을 자동 큐레이션한다. 한국관광공사 TourAPI 4.0을 기반으로 **시간(월별 축제)** 과 **공간(6권역)** 두 축을 교차하여, "이번 달 어디 갈까?" 라는 단 한 번의 고민조차 줄인다.

### 2. 서비스 필요성

- **현대인 의사결정 피로의 해소**: 정보 과잉의 시대, 사용자가 가장 원하는 건 더 많은 데이터가 아니라 **"지금 나에게 맞는 한 가지 답"**. 이달여행은 진입 즉시 큐레이션을 노출하여 의사결정 비용을 0에 가깝게 낮춘다.
- **공공 관광데이터의 사용자 체감 확대**: TourAPI의 풍부한 데이터가 B2B·연구용에 머무르지 않고 일반 사용자가 매월 자연스럽게 접하는 서비스로 재구성될 필요. 축제·관광지·음식점을 한 화면에서 시간축으로 큐레이션해 데이터 활용도를 체감 가능한 수준으로 끌어올린다.
- **국내 여행 실행 전환율 제고**: 시즌 축제는 지역경제 활성화의 핵심 자산. 사용자가 "내가 가면 이런 일정이 나온다"를 즉시 보면 **결정 → 실행** 간 마찰이 줄어들어 실제 방문 전환이 늘어난다.

> **[이미지 삽입 가이드 1 - 기획 배경]** 
> - `home_regional_filters_all_1776177509993.png` (전국 축제 큐레이션 화면)
> - `home_regional_filters_gangwon_1776177556022.png` (권역별 필터링 화면)
> *설명: 진입 즉시 "이번 달 어디 갈까?"의 답을 제시하는 메인 화면*

## 2) 서비스 개요

### 1. 기획 서비스 소개 (한 줄)

**"고민하지 마세요. 이번 달은 여기 가세요." — 시즌 축제 기반 AI 여행 코스 자동 큐레이션 모바일 웹**

### 2. 기획 서비스 주요 기능

1. **진입 즉시 시즌 축제 피드**: 사용자가 별도 입력 없이 사이트에 진입하면 이번 달의 축제를 전국·권역별로 자동 큐레이션
2. **축제 상세 + 주변 탐색**: 선택한 축제 주변 관광지/맛집을 반경 기반으로 즉시 제시
3. **AI 코스 자동 생성**: 선택한 장소들을 Gemini AI가 이동시간·체류시간 고려해 하루 또는 1박2일 타임라인으로 조합
4. **경량 경로 엔진**: 자체 하버사인 연산으로 외부 API 지연 없이 즉시 코스 렌더링
5. **카카오맵 내비 직연결**: 완성된 코스에서 각 장소를 카카오맵 공식 URL Scheme으로 바로 길안내 연결

> - `ai_button_disabled_zero_spots_1776177957248.png` (최소 탐색 조건 방어 UI)

### 3. 서비스 차별성

본 서비스의 핵심 독창성은 **"진입 즉시 가치 제공" + "시간축 × 공간축 2D 큐레이션"** 두 가지다.

**(1) 진입 즉시 가치 제공**
대부분의 여행 정보 서비스는 사용자가 지역·기간·테마를 입력하고 검색해야 결과를 받는다. 이달여행은 사용자의 별도 입력 없이 **사이트 진입 순간 "이번 달 갈 만한 곳" 큐레이션을 즉시 노출**한다. 의사결정 피로가 큰 현대인에게 "고민할 필요 없이 바로 보여주는" 진입 경험은 강력한 차별 가치다.

**(2) 시간축 × 공간축 2D 큐레이션**
- 공간 중심 서비스: "서울 맛집 BEST", "부산 관광지 추천" — 지역은 있으나 "언제"라는 시간 맥락이 약함
- 시간 중심 서비스: 일반 캘린더·이벤트 나열 — 지역별 동선 설계 부재
- **이달여행**: 월별(시간) × 6권역(공간)의 교차 큐레이션 + AI가 동선까지 완성 → "이번 달 어디, 어떻게"의 답을 한 번에 제시

추가 강점:
- **공개 API 생태계 기반 재설계**: OpenStreetMap(Leaflet) 기반 지도 렌더링과 카카오맵 공식 URL Scheme 연동으로 외부 토큰 의존도를 최소화. 장기적 확장성·운영 안정성 확보.
- **즉시 응답성**: 자체 하버사인 연산으로 네트워크 왕복 없이 코스 생성 순간 렌더링. 모바일 환경 UX 최적화.
- **심리스한 내비 연동**: 카카오맵 공식 공개 URL Scheme으로 별도 앱 설치 없이 실제 이동 단계 연결.

### 4. (지역 특화 관련 사항)

본 서비스는 전국 단위를 대상으로 하되 **6권역(서울·경기, 강원, 충청, 전라, 경상, 제주)** 별 섹션 구조를 적용한다. 각 권역의 TourAPI areaCode를 체계적으로 매핑하여 지역성 있는 큐레이션을 제공하며, 향후 1단계 확장 시 **강원·제주** 등 축제 밀도 높은 권역에 대한 심화 큐레이션을 우선 구축할 계획이다.

## 3) 데이터 활용 방안 *한국관광공사 OpenAPI 활용 필수

### 1. 활용 예정 한국관광공사 OpenAPI

**기반 API (TourAPI 4.0 / KorService2)**

| API | 용도 | 활용 엔드포인트 |
|---|---|---|
| 행사정보조회 | 월별·지역별 축제 피드 생성 | `searchFestival2` |
| 키워드검색 | 축제 주변 관광지/맛집 탐색 | `searchKeyword2` |
| 공통정보조회 | 장소 상세 페이지 렌더링 (개요·대표이미지) | `detailCommon2` |
| 위치기반관광정보 | 현재 위치·축제 좌표 기반 반경 탐색 | `locationBasedList2` |

### 2. 데이터 활용 방식

**진입 → 자동 큐레이션 → 매시업 흐름**

1. 사용자 사이트 진입 → 현재 월·전국 기준으로 `searchFestival2` 자동 호출하여 **이번 달 축제 피드 즉시 노출** (사용자 입력 불필요)
2. 사용자가 권역 또는 특정 축제 선택 → 좌표(`mapx`, `mapy`)를 키로 `searchKeyword2` · `locationBasedList2` 조합 호출로 주변 관광지(contentTypeId 12)와 음식점(39) 수집
3. 사용자가 최종 동선에 포함할 장소를 선택 → `detailCommon2`로 각 장소 개요·대표이미지를 로드해 카드 UI 구성
4. 선택된 장소 배열을 Gemini AI 프롬프트에 전달 → 하루 or 1박2일 타임라인(JSON) 자동 생성

**데이터 정합성·안정성 처리**
- TourAPI `eventStartDate`·`eventEndDate` 날짜 범위 기반 필터링으로 종료된 축제 자동 제외
- `areaCode` 매핑 테이블을 프런트엔드 상수로 관리(`REGION_AREA_CODES`) → 권역 확장 시 상수만 수정
- 이미지 로딩 실패·데이터 공란에 대한 폴백 UI 구축
- OpenStreetMap 타일을 기본 지도로 사용하되, 경로 안내는 카카오맵 공식 URL Scheme으로 위임하여 외부 토큰 의존도 최소화

> **[이미지 삽입 가이드 3 - 길안내 연동 및 위치 기반 기능]**
> - `use_my_location_result_1776177748167.png` (현재 위치 기반 길안내 시작점 설정)
> - `origin_autocomplete_seoul_1776177678415.png` (출발지 검색 및 자동완성)

## 4) 서비스 발전 방향

### 1. 개발 서비스 향후 발전방향

**단기 (출시 ~ 3개월)**
- 지역 RTO(Regional Tourism Organization) 제휴를 통한 공식 축제 데이터 검증 루프 구축
- 사용자 피드백 기반 코스 품질 개선 (Gemini 프롬프트 튜닝 + 사용자 평가 수집)
- 오픈 그래프 기반 공유 기능 (SNS 유입 활성화)

**중기 (3~12개월)**
- OSRM(OpenRouteService) 연동 옵션 추가 → 하버사인 직선거리를 실제 도로 기반 시간으로 보강
- 숙박(TourAPI 32번 타입) 통합 → 1박2일 코스에 실제 숙소 추천까지 확장
- 공식 매거진(KTX 매거진 등) 및 축제 상세 콘텐츠 채널과의 협력을 통한 OpenAPI 외 데이터 통합(프로그램 일정·주변 정보·검증된 큐레이션 등) — 매거진 라이프사이클 연장 + 코스 품질 강화 윈윈 모델 검토
- 다국어 지원 (일·영·중) — 한국관광공사의 다국어 TourAPI 연동으로 인바운드 관광객 대상 확장

**장기 (1년 이상)**
- B2B API 제공: AI 코스 생성 엔진을 외부 여행사·플랫폼에 라이선스
- 지역 축제 주최 측 B2G 제휴: "이달여행 공식 등재 축제" 라벨링으로 지자체 관광 활성화 연계
- 해외 오픈 관광 데이터 연동: 공개 API 생태계 기반이라 해외 확장 비용 구조 유리

**지속 가능성**
- 클라우드 비용: Vercel Edge 기반 정적 배포 + 서버리스 함수로 트래픽당 비용 극소화
- 데이터 비용: 공공 OpenAPI 사용으로 기본 데이터 수집 비용 0원
- 운영 체계: 자동화된 데이터 갱신 파이프라인으로 1인 운영 가능

> - `verify_day2_tab_1776178614844.png` (2일차 코스 검증 화면)
> *설명: 생성된 코스의 거리와 소요 시간이 실제 이동 가능한지 검증하는 UI*
