## 🎯 코어 세그먼트: 가족 단위 여행자 (30~45세, 자녀 동반)

---

### 1. Who (누구)

**페르소나: "주말 여행 결정자 부모"**

- 나이: 30~45세 (1인 결정자 또는 부부 공동 결정)
- 가족: 자녀 1~2명 (3~12세)
- 직업: 직장인·자영업·프리랜서
- 라이프스타일:
  - 주말·연휴 가족 시간 = 우선
  - 모바일 = 검색·결정 채널
  - 인스타·블로그 = 영감
  - 예산 = 1박 2일 ~50만원 또는 당일치기 ~10만원
- 결정 패턴:
  - 주중 (목·금) 검색 시작
  - 주말 직전 (금요일 저녁) 최종 결정
  - 검색 시간 = 평균 30분~1시간

---

### 2. Why (왜)

**Job-to-be-Done**: "주말에 가족과 함께 의미 있는 시간을 보내고 싶다."

**기능적 Job**:
- 갈 만한 곳 찾기
- 동선 짜기
- 식사·간식 장소 정하기
- 자녀 친화 검증

**감정적 Job**:
- "좋은 부모"·"가족 시간 잘 챙기는 부모" 자기 이미지
- 주말 공허함 vs 추억 만들기 균형

**사회적 Job**:
- 인스타·SNS 자녀 사진·여행 사진
- 가족 단톡방·친척 모임에서 "이번 주말에 어디 갔다 왔어"

---

### 3. What Before (기존 상황)

**현재 어떻게 해결?** (JTBD의 "before")

```
주중 (목·금)
  ↓ 네이버·다음 검색 ("주말 가족 여행지" "당일치기 [지역]")
  ↓ 블로그 5~10개 읽기 (광고·체험단·후기 혼재)
  ↓ 한국관광공사 사이트 (정보만 노출, 코스 X)
  ↓ 인스타 #가족여행 검색
  ↓ 부부 상의 (어디 갈지, 점심·간식 어디서)
주말 (토)
  ↓ 출발
  ↓ 현장에서 "여기 다음에 어디?" 즉흥 결정
  ↓ 가끔 자녀 만족도 낮음 (지루·피로·길)
```

**페인 포인트**:
- 검색 시간 30분~1시간 (의사결정 피로)
- 정보 신뢰도 낮음 (광고·체험단)
- 가족 친화 필터 X (성인 위주 정보)
- 동선 비효율 (멀리 떨어진 장소 무리)
- 축제·이벤트 정보 분산 (놓침)

---

### 4. How (IdalTrip이 어떻게)

**솔루션**:
1. **한 줄 입력** ("아이 두 명 7세·10세, 서울 출발, 일요일 당일치기")
2. **5초 안에 코스** (한국관광공사 API + AI 매칭)
3. **일정 표 + 지도** (이동 시간·점심·간식 자동)
4. **가족 친화 필터** (자녀 연령·접근성·소요 시간)
5. **수정 가능** (빼고 싶은 항목 X, 다른 옵션 자동 제안)
6. **공유 기능** (URL → 부부·친척 단톡방)

---

### 5. What After (이후 상황)

**JTBD의 "after"** — 사용자 만족 상태

```
목요일 저녁
  ↓ IdalTrip 한 줄 입력 (1분)
  ↓ 5초 후 코스 받음
  ↓ 부부 같이 본다 → "여기 빼자, 여기 추가" 1~2번 수정 (5분)
  ↓ 공유 URL → 친척에 보여줌
주말 (토·일)
  ↓ 일정대로 출발
  ↓ 자녀 만족도 ↑ (가족 친화 필터)
  ↓ 축제·이벤트 놓침 X (자동 추천)
  ↓ 동선 효율 (이동 시간 최적화)
주말 끝
  ↓ "다음 주말에도 IdalTrip 써야지" → 재방문
  ↓ 인스타 #IdalTrip 또는 코스 공유 → 친척·친구 확산
```

**감정**:
- 검색 부담 ↓
- "좋은 부모" 자기 이미지 ✅
- 가족 시간 만족도 ↑
