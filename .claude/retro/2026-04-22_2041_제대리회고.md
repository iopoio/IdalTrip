## 무엇을 했나
Kakao API 심사 반려(403)로 인해 막힌 기능을 OpenStreetMap(Leaflet)과 TourAPI 기반으로 긴급 전환하여 서비스의 블로커를 제거했습니다.

## 어디서 막혔나 (또는 예상보다 빨랐던 부분)
`npm run dev` 로컬 QA 시, Vercel Serverless Function으로 구동되는 `/api/generate-course` 엔드포인트가 로컬 환경에 구성되어 있지 않아 코스 생성 결과 렌더링을 완전히 테스트하지 못한 부분이 아쉬웠습니다. 하지만 Kakao 의존성 제거 자체는 예상보다 수월하게 진행되었습니다.

## 다음엔 어떻게 할 건가
외부 벤더 종속성이 강한 기능(지도, 내비 등)은 심사 거절이나 장애 시에 대체할 수 있는 Fallback 전략(예: OSM 타일 사용 등)을 미리 설계해두면 좋겠다는 것을 배웠습니다.
