## 2026-04-17 저녁 (18시) — Phase5 지시서 복구 오판정 반복

### 클과장
**무엇을 틀렸나 (3건, 같은 패턴 반복)**
1. 제대리가 `제대리_Phase5_지시서.md` 찾는다는 소식에 파일명만 매칭해서 "이미 완료된 파일, 제대리 할 일 없음" 판정 (C3 +1회)
2. 원본 지시서 복구 후 각 Task를 grep으로만 확인 → "Task 3, 6, 7 미구현"으로 잘못 판정. 실제로는 HomePage에 `REGIONS`/`travelDate`/`navigate('/places')` 전부 있었음. `"어디로 떠날까요?"` 같은 원 지시서 문구만 찾아서 못 봄 (C3 +1회)
3. 실측 안 하고 원 지시서의 `ExploreResultPage`/`/explore` 이름이 현재 코드에 없는 걸 "미구현"으로 치부. 실제로는 이름만 `PlaceSelectionPage`/`/places`로 바뀐 것. 기능은 있음 (C3 +1회)
4. 후추님이 "실제로 다 구현된거야?" 한 번 물어본 뒤에야 정밀 검증 시작. 그 전에 후추님이 두 번이나 "이거 아닐까?" 힌트 줬는데 놓침

**왜 틀렸나**
- 검증을 "키워드 grep"이나 "파일명 존재"로 축소 시각화
- 기능 수준 검증(실제 UI 플로우가 동작하는지)이 아니라 텍스트 수준 검증
- 원 지시서의 변수명/라우트명이 바뀔 수 있다는 전제 없이 원문 그대로 매칭

**다음엔**
- 기능 검증: grep 대신 파일 전체 read + 기능 단위 매핑
- 이름이 바뀌었을 수 있으니 기능(routing, state, handler)으로 검색
- 후추님 "정말이야?" 한 번 들어오면 바로 정밀 재검증. 두 번 들으면 이미 늦음

- 프로젝트 리뷰 초반에 인프라 스캔(git/배포) 루틴 없음

**다음엔**

- 내가 수정 안 한 파일 섞여있으면 `git add -p` 또는 1회 후추님 확인
- 프로젝트 리뷰 시작 시 `git rev-parse` + 배포 URL 유무 먼저 체크
