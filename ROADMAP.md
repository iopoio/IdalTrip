## 크리티컬 패스

```
스티치 디자인 수령 (1-1)
    → 디자인 적용 (1-2~1-4)
        → UI 구현 (2C)

TourAPI 관광지/맛집 (2A-2) + Gemini 프롬프트 (2B-2)
    → 코스 생성 엔진 (2B-1~2B-6)
        → 결과 페이지 (2C-4~2C-6)
            → E2E (3-1) → 배포 (3-6) → 제출 (4-3)
```

병렬 가능: Phase 1 디자인 적용 ↔ Phase 2A 데이터 레이어는 동시 진행 가능
블로커: 스티치 디자인이 늦으면 UI 작업 전체가 밀림
